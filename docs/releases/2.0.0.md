# AriaNg 2.0.0

AriaNg 2.0.0 新增了更具韧性的 BitTorrent 小文件过滤流程，并简化了重复任务的创建体验。

## 主要更新

- 新增可选的全局规则：对新建磁力链接和种子任务，可按可配置的大小阈值排除小于阈值的文件。
- 新增可持久化且可恢复的 BT 过滤任务机制，支持 RPC 隔离、元数据子任务发现、重试保护、失败回退恢复和清晰的处理结果摘要。
- 在新建任务区域旁新增桌面端下载路径快捷入口。
- 可按 RPC 端点记忆最近一次提交的下载路径，并支持从既有全局路径历史迁移。
- 保持桌面端快捷入口、高级选项字段、手动输入与历史选择之间的状态同步。
- 优化响应式工具栏行为：桌面端长状态文本可读，搜索框会在换行前隐藏，移动端过滤控件可使用完整可用宽度。
- 修复移动端工具栏换行时固定头部间距动态计算问题。
- 防止基础任务刷新抢占正在进行中的完整文件详情刷新。

## 可靠性与发布工程

- 为 BT 元数据恢复、RPC 切换、队列持久化、回退行为、路径记忆、工具栏布局和任务列表刷新顺序新增了广泛的回归测试覆盖。
- 将 GitHub 发布工作流升级到 Node.js 24。
- 发行归档继续提供标准版与 All-In-One 两种格式。

## 兼容性与安全说明

AriaNg 仍是基于 AngularJS 1.x 的应用，并使用 Bootstrap 3 与 AdminLTE 2。依赖扫描器会在该遗留技术栈及其传递依赖中报告已知安全告警；这些传递依赖并不会直接打包进最终生成的静态归档。请务必通过认证和可信网络访问来保护 aria2 RPC 端点。要彻底消除这类遗留依赖风险，仍需在未来进行框架迁移。
