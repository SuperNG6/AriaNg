# AriaNg 2.0.0

AriaNg 2.0.0 adds a resilient BitTorrent small-file filtering workflow and streamlines repeated task creation.

## Highlights

- Added an optional global rule for new magnet links and torrent tasks that excludes files below a configurable size threshold.
- Added persistent, recoverable BT filtering jobs with RPC isolation, metadata-child discovery, retry protection, fallback restoration, and clear processing summaries.
- Added a desktop download-path shortcut beside the new-task controls.
- Remembered the most recently submitted download path per RPC endpoint, with migration from the existing global path history.
- Kept the desktop shortcut, the advanced option field, manual edits, and history selections synchronized.
- Improved responsive toolbar behavior: long processing status remains readable on desktop, search hides before it wraps, and mobile filter controls use the full available width.
- Fixed dynamic fixed-header spacing when the mobile toolbar wraps.
- Prevented basic task refreshes from overtaking an in-flight full file-detail refresh.

## Reliability and release engineering

- Added extensive regression coverage for BT metadata recovery, RPC switching, queue persistence, fallback behavior, path memory, toolbar layout, and task-list refresh ordering.
- Updated the GitHub release workflow to Node.js 24.
- Release archives continue to ship in standard and All-In-One formats.

## Compatibility and security note

AriaNg remains an AngularJS 1.x application using Bootstrap 3 and AdminLTE 2. Dependency scanners report known advisories in this legacy platform and in transitive packages that are not included in the generated static archives. Protect the aria2 RPC endpoint with authentication and trusted network access. A future framework migration is required to remove the remaining legacy dependency risk completely.
