# 新建 BT 任务小文件过滤功能设计

## 背景与目标

用户一次可能提交几十或上百个磁力链接。当前 aria2 默认选择 BT 任务中的全部文件，用户只能在下载后手动删除不需要的小文件，浪费带宽和磁盘空间。

本功能在 AriaNg 新建任务时按文件大小自动生成 aria2 的 `select-file` 选项。它必须保持 AriaNg 的纯静态前端部署方式，不修改 aria2，也不要求安装服务端脚本。关闭页面时未完成的过滤工作保存在浏览器；再次打开同一 AriaNg 后自动继续。

## 范围

支持：

- 批量粘贴的磁力链接；
- 本地上传的 `.torrent` 文件；
- URL 路径以 `.torrent` 结尾的 HTTP/HTTPS 远程种子文件（查询参数和片段不参与路径判断）；
- “立即下载”和“稍后下载”两种提交方式；
- 同一批次中混合 BT 任务、普通文件直链的情况。

不支持：

- 普通 HTTP、FTP、SFTP 文件的大小过滤；
- Metalink 文件过滤；
- AriaNg 页面关闭期间在浏览器外继续执行过滤；
- 跨浏览器、跨设备或清除站点数据后的队列恢复；
- 在服务端安装常驻过滤程序。

## 用户界面

过滤器只在 `/new` 页面显示，位于主工具栏帮助按钮右侧。离开新建页面后，该位置恢复现有行为：三个任务列表页显示“文件列表”，其他页面不显示过滤器。

采用始终可见的直接输入布局：

> ☑ 排除 BT 任务中小于 `[100]` MB 的文件

界面文案统一使用“排除小文件”描述不下载任务内文件的行为，避免“过滤”被误解为隐藏任务或筛选列表。统计结果必须明确以“任务”为单位；无需排除或发生容错时统一描述为“保留全部文件”。

交互规则：

- 首次使用默认关闭，阈值预填 `100`，不改变现有默认下载行为；
- 勾选状态和阈值保存到 AriaNg 本地设置，并在下次打开时恢复；
- 关闭过滤时保留输入值，但将输入框置灰；
- 阈值只接受 `1` 到 `102400` 的正整数；
- 输入无效时禁止提交任务，并显示输入错误状态；
- 文件大小严格小于阈值时排除，等于阈值时保留；
- 提交任务时保存阈值快照，后续修改工具栏阈值不影响已提交任务；
- 控件需要适配浅色、深色和窄屏布局。

控件提示说明：`仅适用于新建的磁力链接和种子任务。任务同时包含大、小文件时，将排除小于设定值的文件；若全部文件都小于设定值，则保留全部文件。任务完成后，aria2 可能删除被排除的文件及下载目录中的同名文件。`

### 过滤控件视觉层次

采用“轻量分组”方案，在不改变 AriaNg 工具栏骨架的前提下，把复选框、完整说明、阈值输入框和单位组织为一个独立规则容器：

- 启用时使用白色表面、左侧细蓝色强调边、轻微阴影和小圆角，与工具栏灰色背景形成清晰但克制的分层；
- 数字输入框保留独立边框和轻微内阴影，明确它是可编辑区域；
- 关闭时移除蓝色强调和外阴影，容器、文字和输入框统一降为中性灰，但继续显示上次阈值；
- 输入无效时，容器左侧强调边和输入框同步使用错误色，并在控件旁显示 `请输入 1–102400`，不能只依靠颜色表达错误；
- 深色主题中，规则容器比工具栏背景提升一级，输入框使用更深的内层表面，并保持足够的文字和边框对比度；
- 窄屏时把规则容器作为整体自然换到工具栏第二行，不截断完整句子，也不挤压主要任务操作按钮；页面不得产生水平滚动条；
- 控件仍使用现有复选框与 number 输入行为，不增加弹层、下拉菜单或新的图标交互。

### 工具栏过滤状态区

在上下文按钮之后、右侧搜索框之前增加过滤状态区，也就是任务列表页“文件列表”按钮右侧的空白位置。它可以在所有页面显示当前后台过滤状态；没有活动任务和待展示结果时不占用空间。

状态文案按阶段变化：

- 恢复队列：`正在恢复 28 个尚未完成小文件处理的 BT 任务`；
- 正在处理：`正在排除 BT 任务中的小文件：已处理 7/28 个任务`；
- 等待文件列表：`正在等待 21 个 BT 任务的文件列表`；
- 全部完成：短暂显示 `BT 小文件处理完成：72 个任务已排除小文件，6 个任务保留全部文件` 后自动隐藏；
- 存在回退：使用警告色显示 `1 个 BT 任务无法排除小文件，已回退为保留全部文件` 后自动隐藏。

活动状态前显示小型旋转图标，完成使用成功色，回退使用警告色。桌面端显示完整文案；窄屏按状态分别显示 `恢复 28`、`等待 21`、`处理中 28`、`已处理 78` 或 `已回退 1`，完整内容通过标题提示查看。状态区仅展示汇总，不新增详情页或点击操作。

## 核心规则

对取得的完整文件清单计算保留索引：

1. 大小大于或等于阈值的文件保留；
2. 大小小于阈值的文件排除；
3. 全部文件均达到阈值时，不修改文件选择；
4. 全部文件均低于阈值时，视为“任务本身由小文件组成”，必要时恢复全部文件选择后完整下载；
5. 只有同时包含阈值上下两类文件时才真正执行过滤。

全大文件表示阈值没有排除任何文件，因此协调器不得改写任务原有的 `select-file` 或
`bt-remove-unselected-file`；若用户已做过部分选择，应原样保留。全小文件则是明确的安全回退，
必须避免继承部分选择而违背“完整下载”约定。

成功过滤时，在一次 `aria2.changeOption` 请求中设置：

- `select-file=<保留文件索引>`；
- `bt-remove-unselected-file=true`。

自动清理只对本功能确实排除过文件的任务启用，不修改 aria2 全局设置。全小文件、全大文件、非 BT 和过滤失败回退的任务均不会由本功能新启用自动清理；恢复全选或失败回退时保留任务原有的清理值。

## 架构

### 工具栏与新建任务控制器

主控制器负责按路由显示过滤控件，设置服务负责读取和保存启用状态、阈值。`NewTaskController` 在提交时读取设置并为每个任务创建不可变的过滤意图，包括阈值字节数和原始提交方式。

未启用过滤时，新建任务沿用原流程。启用时：

- URI 任务附加 `pause-metadata=true`。普通直链不受该选项影响；磁力和远程种子产生的后续下载保持暂停；
- 本地 `.torrent` 使用暂停方式创建；
- 成功创建后，把根 GID 和过滤意图交给协调器。

### BT 文件过滤协调器

新增单一职责服务管理异步过滤，不把轮询、恢复和重试堆入控制器。协调器在应用启动并连接 RPC 后运行，因此用户重新打开任意 AriaNg 页面都可恢复队列。

每个队列项目至少保存：

- 当前 RPC 配置的稳定标识；
- 根 GID 和已发现的 BT 子任务 GID；
- 阈值字节数；
- 原始操作是“立即下载”还是“稍后下载”；
- 当前阶段、重试次数、最近错误和更新时间。

队列保存在浏览器本地存储，并按 RPC 配置隔离。稳定标识固定由
`protocol|host|port|interface` 组成，刻意不包含 RPC secret 或自定义请求头；它标识 aria2
端点而不是凭据/路由配置。切换 RPC 服务时只处理属于当前服务的项目，避免不同 aria2 实例之间 GID 碰撞。

## 数据流

### 磁力和远程种子

1. aria2 运行元数据下载；
2. `pause-metadata=true` 使元数据生成的实际 BT 子任务暂停；
3. 协调器查询根任务，沿 `followedBy` 找到子任务；
4. 子任务出现 `bittorrent` 和完整 `files` 后计算选择；
5. 需要过滤时提交 `select-file` 与自动清理选项；
6. 全小文件时必要时恢复全部文件选择后完整下载；全大文件时不修改已有文件选择，因为阈值没有排除任何文件；
7. 原始操作为“立即下载”时执行 `unpause`，为“稍后下载”时继续暂停。

### 本地种子

1. 暂停调用 `aria2.addTorrent`；
2. 使用返回的 GID 读取文件清单；
3. 按相同规则应用过滤；
4. 恢复原始“立即下载/稍后下载”状态。

### 普通直链

协调器确认任务不是 BT 后将其标记为不适用。普通直链保持原有启动状态，不参与过滤统计中的成功数量。

## 关闭页面与恢复

过滤完成并已启动的任务由 aria2 独立运行，关闭 AriaNg 不影响下载。尚在等待元数据的任务会在 aria2 中继续获取元数据，但实际 BT 子任务保持暂停。

重新打开 AriaNg 并连接同一 RPC 服务时，协调器：

1. 读取本地待处理队列；
2. 查询根 GID 和已保存的子 GID；
3. 以 aria2 当前真实状态为准进行对账；
4. 跳过已经完成的步骤；
5. 继续过滤并按原始操作启动或保持暂停。

恢复依赖同一浏览器的站点数据、同一 RPC 服务，以及任务或相关停止记录未被用户清除。发现恢复任务时，在工具栏过滤状态区显示恢复进度。

## 容错与回退

- 元数据尚未取得不是过滤错误；任务保持等待，不设置固定的元数据超时；
- RPC 暂时断开时保留队列，重连后继续；
- 应用过滤失败时短间隔重试 3 次，并在每次重试前重新读取任务状态；
- 网络结果不确定时必须先对账文件选择状态，避免重复操作或错误启动；
- 用户已删除任务时静默移除队列项目；
- 协调器以 250ms 节拍串行处理一个任务，只有存在当前端点的待处理任务时才发起 RPC；在 RPC 响应零延迟的理想条件下，100 个已就绪任务可在 100 个节拍（约 25 秒）内各调度一次，这只是理想调度容量，实际耗时取决于 RPC 延迟，并非完成时间上限；
- 根 GID 已不存在且连续 3 次成功扫描等待队列都没有发现唯一子任务时，视为用户已删除并静默移除；RPC 失败不计数，多个候选子任务保持等待。三次扫描语义不变，但在队列中只有该任务时最快约 750ms 完成；
- 远程种子根任务进入终态但尚未报告子任务时，同样保留连续 3 次成功扫描的子任务创建竞态窗口，之后按根任务终态确定性结算；
- 最终确认无法过滤时，恢复或确认全部文件选择；“立即下载”任务正常完整下载，“稍后下载”任务保持完整任务暂停；
- 回退任务恢复任务原有的 `bt-remove-unselected-file` 值，不得由本功能新启用它，并显示非阻塞警告；
- 某个任务失败不阻塞同批其他任务。

批量反馈使用工具栏状态区和必要的警告通知，避免产生大量弹窗，例如：

> BT 小文件处理完成：72 个任务已排除小文件，6 个任务保留全部文件，21 个任务等待文件列表，1 个任务已回退。

仍在等待元数据的任务完成处理后更新工具栏汇总。只有需要用户注意的最终回退才同时触发非阻塞警告通知；单个失败可显示任务名，名称不可用时显示 GID。

## aria2 限制

BT 文件可能共享 piece。即使文件未被选择，aria2 仍可能为边界 piece 创建空文件或写入少量数据。任务正常完成后，`bt-remove-unselected-file=true` 会删除未选择文件，但任务暂停、失败或被强制停止时残留可能暂时存在。本功能不能保证过滤期间磁盘上绝对不出现这些文件。

## 测试与验收

自动化测试覆盖：

- 阈值边界、全小、全大和混合文件清单；
- 磁力根任务到 `followedBy` 子任务的转换；
- 本地种子、远程种子、普通直链与混合批量；
- 立即下载和稍后下载；
- RPC 重试、断线恢复、页面重载、RPC 服务隔离和删除任务；
- 成功过滤时自动清理，回退时不误删；
- 设置持久化、无效输入和现有新建任务回归；
- 工具栏恢复、处理中、等待、完成和回退状态；
- 浅色、深色、桌面和窄屏工具栏样式。

验收标准：

1. 混合大小的 BT 任务在下载文件数据前完成选择；
2. 全小文件任务完整下载；
3. 关闭并重新打开同一浏览器后可继续待处理过滤；
4. 过滤成功后，立即任务启动、稍后任务暂停；
5. 最终过滤失败时完整下载且不误删；
6. 普通直链、Metalink 和未启用过滤时的行为不变。
