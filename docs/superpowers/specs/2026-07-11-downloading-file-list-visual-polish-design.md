# 主下载页文件列表视觉精修设计

## 1. 文档信息

- 产品：AriaNg
- 功能：主下载页文件列表
- 精修范围：任务内嵌文件列表的边缘、背景和视觉层级
- 生效页面：`/downloading`
- 文档日期：2026-07-11
- 状态：已确认，待实施

## 2. 背景与问题

主下载页文件列表功能已经实现，但当前嵌套区域主要依靠左右缩进和一条顶部边线表达层级。实际界面中存在以下问题：

- 文件列表没有完整边界，右侧和底部看起来像被截断。
- 任务摘要、文件表头和文件行的背景层级接近，归属关系不够明确。
- 外层任务表和内层文件表同时继承 `.task-table` 的斑马纹及悬停样式，产生视觉叠加。
- 纯 `30px` 缩进占用了空间，却没有形成足够清楚的任务与文件从属关系。
- 文件列表与下一任务之间缺少稳定的视觉停顿。
- 已禁用的文件复选框视觉强度接近可交互状态。

根因是嵌套文件区直接复用了通用 `.task-table` 结构，但只补充了顶部边框和间距，没有为嵌套场景建立独立、完整的容器层级与主题覆盖。

## 3. 设计目标

- 用户能够一眼识别每组文件属于哪个下载任务。
- 任务摘要、文件表头、文件行和下一任务之间形成清晰但克制的四级层次。
- 保持 AriaNg 现有高密度、工业工具型界面，不引入大圆角、重阴影、动画或全新视觉语言。
- 浅色、深色和窄屏布局均具有完整边缘和可读对比度。
- 只精修视觉，不修改文件数据、刷新、折叠、排序或任务操作逻辑。

## 4. 选定方向

采用“A · 内嵌面板”方案：

- 文件列表被完整细边框包围。
- 左侧使用 AriaNg 蓝色强调线表示文件对上方任务的从属关系。
- 文件表头使用独立背景。
- 文件行使用明确的行分隔和局部悬停状态。
- 使用小圆角和极浅阴影建立层次，但不改变主任务列表的整体风格。

未采用以下方向：

- 左侧层级轨道：视觉更轻，但文件区右侧和底部边界仍然偏弱。
- 完整任务卡片：层级最强，但会明显改变 AriaNg 连续任务列表的既有风格。

## 5. 浅色主题规范

### 5.1 内嵌面板

- 保持在所属任务行内部。
- 左右内缩从当前 `30px` 调整为更紧凑的 `15px`，让面板更接近任务内容列宽。
- 与任务摘要之间保留 `8px` 间距。
- 展开状态下任务行底部内边距为 `12px`，使面板和下一任务边线不粘连。
- 使用完整 `1px solid #d6dfe6` 边框。
- 左边框使用 `3px solid #208fe5`。
- 圆角为 `3px`。
- 使用极浅阴影 `0 1px 2px rgba(32, 52, 68, 0.05)`。
- 面板内容溢出隐藏，保证表头和行背景不越过圆角。

### 5.2 文件表头

- 背景使用浅灰蓝 `#eef4f8`。
- 底部使用 `1px solid #dce4ea` 分隔线。
- 保留现有 `12px` 字号和三列布局。
- 表头上下内边距统一为 `6px`，与文件行区分但不增加过多高度。
- 排序图标和文字继续沿用现有颜色及交互。

### 5.3 文件行

- 所有文件行使用白色背景，覆盖从外层 `.task-table` 继承的奇偶斑马纹。
- 行间使用 `1px solid #e6ebef` 分隔线。
- 最后一行不显示底部分隔线，避免与面板底边重复。
- 行上下内边距保持 `6px`。
- 行悬停背景使用淡蓝灰 `#f6fafc`，只作用于当前文件行。
- 文件区悬停不得触发或改变所属任务的选中状态。

### 5.4 只读状态

- 文件与目录复选框继续保留，以保持与任务详情文件列表一致。
- 已禁用复选框和对应标签整体透明度调整为 `0.72`。
- 鼠标指针保持默认状态，不呈现可编辑暗示。
- 目录展开图标保持正常强度和可点击状态，不随禁用复选框一起降低透明度。

## 6. 深色主题规范

- 面板背景使用 `#1d2226`。
- 完整边框使用 `#3b4650`。
- 左侧强调线继续使用 `#208fe5`。
- 文件表头背景使用 `#242d34`。
- 表头底部分隔线使用 `#3b4650`。
- 文件行背景统一为 `#1d2226`，覆盖内层斑马纹。
- 文件行分隔线使用 `#303840`。
- 文件行悬停背景使用 `#26313a`。
- 文字、进度条和低进度文字颜色继续继承现有深色主题规则。
- 深色主题阴影使用 `0 1px 2px rgba(0, 0, 0, 0.18)`，不使用明显发光效果。

## 7. 任务层级与间距

- 不把整个任务改造成独立卡片。
- 不改变任务行原有名称、进度、速度和右箭头布局。
- 文件面板作为任务行最后一个全宽区域。
- 展开文件面板的任务行增加底部内边距，确保面板底边与下一任务顶部边线之间存在视觉停顿。
- 文件面板关闭时，任务行间距必须与原版一致。
- 搜索和任务排序时，面板继续随所属任务一起隐藏或移动。

## 8. 窄屏规范

- 在 `max-width: 767px` 下，文件面板左右内缩调整为 `5px`。
- 保留完整边框、左侧强调线和圆角。
- 文件表头继续遵循项目现有规则隐藏，不额外增加移动端表头。
- 文件名、进度和大小继续按现有 Bootstrap 列规则堆叠。
- 文件大小保持右对齐。
- 不增加横向滚动条或固定最小宽度。

## 9. DOM 与 CSS 边界

- 为外层任务行增加仅在文件面板可见时生效的语义 class `has-downloading-file-list`。
- 嵌套容器继续使用 `.downloading-file-list`，内部增加或复用清晰的面板 class，避免使用宽泛选择器影响任务详情页。
- 所有新增 CSS 必须以主下载页嵌套文件列表 class 为作用域。
- 明确覆盖内层文件行的奇偶背景和悬停背景，不能依赖通用 task-table 选择器的偶然优先级。
- 深色主题覆盖放在 `default-dark.css`，浅色结构和通用间距放在 `task-table.css`；如需浅色主题颜色覆盖，放在 `default.css`。

## 10. 非目标

- 不修改“文件列表”顶部按钮视觉。
- 不改变任务详情页的文件列表。
- 不改变文件列表列宽、排序、目录缩进算法或进度计算。
- 不增加展开/收起动画。
- 不增加任务级单独文件列表开关。
- 不增加新的字体、图标、图片或依赖。
- 不修改 aria2 RPC、刷新频率或任务状态逻辑。

## 11. 验收标准

1. 文件面板在浅色主题下具有完整的上、右、下、左边界。
2. 左侧蓝线能够清楚表达文件列表属于上方任务。
3. 文件表头、文件行和外层任务摘要具有可辨识的背景层级。
4. 内层文件行不再出现由外层 task-table 规则造成的斑马纹叠加。
5. 文件行悬停只影响当前文件行，不改变整个任务背景或选中状态。
6. 面板与下一任务之间存在清楚但紧凑的间距。
7. 禁用复选框明显表现为只读，目录展开图标仍保持可操作。
8. 深色主题具有完整边框、独立表头、行分隔和悬停层级。
9. 窄屏下无溢出、重叠或额外横向滚动。
10. 文件列表关闭时，原主任务列表的间距和背景不变。
11. `/waiting`、`/stopped` 和任务详情页视觉不变。
12. 数据刷新、目录折叠、文件排序、任务选择和任务右键菜单行为不变。

## 12. 验证方式

- 执行 ESLint。
- 生成 Angular 模板缓存。
- 执行标准版构建和 All-in-One 构建。
- 对浅色、深色、桌面和窄屏分别进行截图或浏览器人工比对。
- 使用至少两个相邻下载任务确认任务与文件面板归属关系。
- 使用多级目录任务确认目录缩进、折叠图标和只读复选框层级。
- 验证文件区点击不会切换任务选中状态。
