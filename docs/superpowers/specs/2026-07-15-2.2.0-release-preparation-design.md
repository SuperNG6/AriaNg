# AriaNg 2.2.0 发版前收尾设计

## 文档信息

- 日期：2026-07-15
- 目标版本：`2.2.0`
- 状态：已批准，待实施
- 范围：本 fork 新增的任务文件列表、BT 小文件过滤、发布自动化及其测试和文档

## 目标

把 `master` 当前已完成并经过浏览器、真实 aria2 验证的功能整理成可发布的 `2.2.0` 状态。收尾只解决已经记录的发布阻断项，不新增产品功能，不扩大到上游依赖漏洞或 AngularJS/Gulp 弃用告警。

## 发布阻断项

### 1. 测试生命周期夹具

`getPendingGidStageMap()` 的正式契约是：协调器未启动或已经 `stop()` 时返回空映射，但持久化任务保留。现有 `test/bt-filter-pending-badge.test.js` 中六个映射用例直接读取未启动的服务，仍按旧契约期待徽标，因此失败。

收尾只修测试前置条件：需要验证运行中映射的用例先调用 `start()`；停止状态仍单独断言返回空映射。不回退产品代码中的停止态保护，也不为测试暴露新的服务 API。

### 2. 冻结文案与全部语言

英文源和简体中文已经冻结为本次发布文案。发布前补齐其余九个语言文件：

- 增加缺失的 `format.bt-file-filter.stage.bulk-inspecting`；
- 同步本轮发生变化的批量过滤标题、候选摘要、运行/完成状态和三行确认文案；
- 保留 `{{count}}`、`{{files}}`、`{{threshold}}`、`{{processed}}`、`{{total}}`、`{{filtered}}`、`{{skipped}}`、`{{failed}}` 等占位符名称；
- 每个语言文件与 `defaultLanguage.js` 保持相同的 36 个 BT 过滤键。

翻译采用简洁、产品化表达，不机械复制英文，也不修改已经冻结的英文和简体中文产品含义。

### 3. 版本与发布资料

版本统一更新为 `2.2.0`：

- `package.json`；
- `package-lock.json` 顶层及根 package 版本；
- `.github/workflows/release.yml` 的手动发布默认值；
- `test/release-workflow.test.js` 的版本契约；
- 新增 `docs/releases/2.2.0.md`。

发布说明聚焦用户可见结果：任务列表 Files 视图、自动与批量 BT 小文件过滤、真实 aria2 稳定回读、暂停归属、状态一致性、响应式界面和发布可靠性。内部测试实现和历史调试事故不写入面向用户的说明。

## 遗留项审计

收尾时复查 `.superpowers/sdd/progress.md`、维护 PRD、最新计划和测试输出，把每个已记录事项归入以下一类：

1. 本次发布已完成；
2. 本次必须修复的确定缺陷；
3. 明确接受且不阻塞发布的低概率限制；
4. 上游问题，不在 fork 维护范围。

只实现第 2 类。自动入队首次持久化失败、多标签页并发、极端崩溃窗口等已接受限制继续记录，不为 `2.2.0` 增加事务、锁或新状态机。

## 验证与证据

发布候选必须通过：

```text
npm test
npm run test:i18n-release
npx gulp lint
npx gulp clean build
npx gulp clean build-bundle
git diff --check
```

UI 最终检查使用清洁源码预览和 Playwright，覆盖 375px 浅色/深色、Files 开关、批量 ready/running/complete、自动/批量状态优先级、路由切换和控制台错误。真实 aria2 已完成可逆主路径、协调器暂停/恢复和用户暂停归属验证；发版收尾只做连接与最终状态只读复核，不再次改动真实任务。

构建中的 Browserslist、Node 弃用或旧依赖告警作为上游信息记录，不在本次修复。任何失败命令必须修复或明确阻断发布，不能以“之前通过”代替当前 `2.2.0` 的新鲜证据。

## 验收标准

1. 六个旧徽标用例使用正确的运行中前置条件，停止态契约仍有覆盖。
2. 英文源和全部十个语言文件各有 36 个 BT 过滤键，严格占位符检查通过。
3. 所有版本入口一致为 `2.2.0`，发布说明文件存在，release workflow 契约通过。
4. 全部测试、严格翻译、lint、两种构建和 diff 检查通过。
5. 375px 明暗主题和主要状态组合无溢出、重复状态或控制台错误。
6. 没有提交生成的 `dist/`、RPC 密钥、个人端点或用户未跟踪文件。
7. 不处理上游依赖漏洞，不引入超出本次发布阻断项的新架构。
