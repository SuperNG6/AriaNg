# AriaNg 2.2.0 Release Preparation Implementation Plan

> **For Codex:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans` to implement this plan task-by-task. Use `superpowers:test-driven-development` for behavior/test-contract changes, `superpowers:systematic-debugging` for unexpected failures, and `superpowers:verification-before-completion` before claiming release readiness.

**Goal:** Finish the fork-owned feature work, synchronize the release contract at version 2.2.0, and produce fresh automated and Playwright evidence without changing upstream dependency state.

**Architecture:** Keep the existing AngularJS application and BT-filter state machine unchanged unless a release audit proves a concrete fork regression. Adapt stale regression fixtures to the service lifecycle contract, finish release-time translations, synchronize all version sources, then verify source and bundle builds through the same UI scenarios. Keep browser automation temporary and do not add a new test framework or permanent maintenance surface.

**Tech Stack:** AngularJS 1.6, Node assertion/VM regression tests, Gulp, Playwright controlling Chrome, aria2 JSON-RPC.

**Scope constraints:** Do not update dependencies, run vulnerability remediation, redesign upstream behavior, commit generated `dist/`/`.tmp/`, commit RPC secrets/endpoints, or touch the user's untracked screenshots and `.claude/skills/playwright-cli/` directory. Do not push, tag, publish, or trigger the release workflow.
